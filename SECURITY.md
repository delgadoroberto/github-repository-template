# Security Policy

Thank you for helping improve the security of this project.

Security is a core principle of this repository. If you discover a security vulnerability, please report it responsibly and avoid publicly disclosing the issue until it has been reviewed.

---

## Supported Versions

This repository is actively maintained using the latest available version.

Security fixes and updates are applied to the current version of the project.

---

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability, please:

1. Do **not** publicly disclose the issue.
2. Open a **private security advisory** if GitHub Security Advisories are enabled.
3. If private reporting is unavailable, contact the repository maintainer through GitHub.
4. Include as much detail as possible to help reproduce and understand the issue.

Useful information includes:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested remediation (if known)

---

## Response Process

After receiving a vulnerability report, the maintainer will:

- Acknowledge receipt of the report.
- Validate the reported issue.
- Assess its impact and severity.
- Develop and test a fix if necessary.
- Release the remediation as soon as reasonably possible.
- Credit the reporter when appropriate (unless anonymity is requested).

---

## Responsible Disclosure

Please allow reasonable time for the issue to be investigated and resolved before making any public disclosure.

Responsible disclosure helps protect users and maintain the security of the project.

---

## Security Best Practices

This repository aims to follow secure development practices, including:

- Secure coding principles
- Dependency management
- Automated security scanning
- Secret detection
- Static Application Security Testing (SAST)
- Infrastructure as Code (IaC) security scanning (when applicable)
- Continuous integration security checks

The specific security controls implemented may vary depending on the project's technologies and requirements.

---

## Contact

If you discover a security vulnerability, please use GitHub's private vulnerability reporting feature when available.

If private reporting is not available and the report does not contain sensitive information, please open a GitHub Issue.

Thank you for helping improve the security of this project.
